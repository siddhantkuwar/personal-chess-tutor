# Frontend Checklist

## Before

- Read ground truth
- Run current app and tests
- Capture current screenshot
- Identify user task and C++ contract
- List all states
- Confirm theme and keyboard behavior

## During

- Reuse tokens/primitives
- Keep domain logic out of React
- Implement loading/empty/error/partial states
- Use accessible semantics
- Pair color with text/icon
- Keep board dominant
- Avoid nested cards and fake data
- Connect progress to backend events

## Before completion

- Build/tests pass
- Light, dark, and minimum-width screenshots captured
- Keyboard/focus/overflow verified
- Import and analysis regressions checked
- Contract changes documented
