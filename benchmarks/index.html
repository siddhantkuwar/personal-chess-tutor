Reproducible performance baselines

The benchmark executable uses deterministic synthetic PGN and standard chess positions. Dataset profiles represent 10, 100, and 1,000 games without including private history.

Run

cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build --target pct-benchmarks
./build/pct-benchmarks > benchmark-results.json

Coverage

FEN and PGN parsing, legal moves, make/unmake, perft, square lookup, event serialization/replay, cache hits, and array-versus-bitboard costs are measured in one machine-readable run.

Interpretation

Use targets.json for product latency and memory gates. The candidate bitboard currently keeps the array board as its correctness oracle; production adoption requires a measured end-to-end gain, not a faster isolated lookup.
