Engineering decisions

Array board remains production

The bitboard candidate passes parity and perft, but arm64 make/unmake measured 60.6 ns array versus 236 ns candidate; square lookup measured 0.93 ns versus 9.38 ns. Retain the readable array board and its measured correctness oracle.

Two workers, one historical slot

Two synthetic workers deliver about 1.88× one-worker throughput. Four scale further in isolation, but two is the conservative thermal default and historical work may occupy only one.

Append-only storage

Immutable checksummed events make partial writes, replay, migrations, and derived-state rebuilding testable. Snapshot and compaction complexity is accepted for a trustworthy audit trail.

Bounded FIFO analysis cache

A 50,000-entry default bounds memory. FIFO avoids lock-heavy LRU mutation on reads; diagnostics expose hits, misses, size, capacity, and evictions.

No memory mapping yet

100-record replay measured 0.42 ms and snapshot-backed 10-game repository load 1.58 ms. Mapping lifetime, stale-file, and bounds risks are not justified.

No transformed puzzles

Public CC0 puzzles are matched only to recurring motifs and nearby rating, then checked by two fresh engine processes. Generated transformations remain disabled because tactical equivalence is costly to prove.
