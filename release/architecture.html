Final architecture

The browser renders facts. C++ owns chess, scheduling, coaching, persistence, and recovery.

Priority invariant

Interactive, current-game, then historical queues are checked in that order. With more than one worker, historical work is capped at workers - 1 in both the game scheduler and engine pool. This prevents an old batch from occupying every execution slot.

Failure boundaries

Worker

A failed engine instance is discarded and recreated. Only its request retries.

Job

Cancellation is per job. Shutdown requests every active token before joining threads.

Storage

The event log is authoritative; snapshots and indexes are replaceable accelerators.

Frontend

Disconnect loses no state. Reconnect starts with a full job snapshot.
